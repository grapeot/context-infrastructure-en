# Axioms Index

> **Axioms** are the user's core cognitive filters — deep philosophical principles distilled from personal experience.
>
> Unlike Skills (execution layer, "how to"), Axioms are the thinking layer ("why"), used to inspire deep thought and provide alternative perspectives.

---

## How to Use

### Quick Browse (INDEX)
**When to read the index**:
- Starting a task involving strategic decisions or architecture design
- Wanting to understand the user's "cognitive baseline"
- Needing to find axiom categories relevant to the current task

### Deep Read (Detailed Files)
**When to read the details**:
- The task involves the user's core values or methodology
- Needing to understand an axiom's "deep reasoning" and "application criteria"
- The user explicitly asks to reference their personal philosophy
- Doing reflection-layer work (distilling axioms, deep retrospectives)

### Core Axiom Clusters

These axioms form organic wholes, often invoked together:

- **AI Management Triangle**: A02(Multiplier) + A03(IC→Manager) + A04(Reliability) — foundational mindset for all AI collaboration
- **Cognitive Asset Group**: T05(Cognition is Asset) + A02 + A03 — value repositioning after code cost collapse
- **Verification Loop**: V02(Verifiability) + T07(Isolate-Process-Verify) + V04(Temporal Grounding) — building trustworthy AI systems

### Active Retrieval Triggers

When the user mentions these keywords, proactively consult the relevant axiom:

- "manage AI", "delegate", "delegation" → A03, A04
- "framework choice", "tech selection", "lock-in" → A06, T08
- "verify", "trust", "reliability" → V series, A04
- "priority", "efficiency", "bottleneck" → M03, X03
- "context", "memory", "docs" → A05, T03
